## 1. Admin

### Scope

- Instellingenpagina onder WordPress Instellingen
- Overzicht van geblokkeerde domeinen
- Acties: toevoegen, bewerken, verwijderen, bulk verwijderen
- Import en export

### Taken

- [x] Nieuwe admin menupagina registreren
- [x] Capability checks centraal maken (`manage_options`)
- [x] Overzichtstabel met zoekfunctie toevoegen
- [x] Paging instelbaar maken
- [x] Nonces op alle mutaties toepassen

### Klaar wanneer

- [x] Admin kan domeinen volledig beheren zonder page reload errors
- [x] Alle admin-acties geven duidelijke succes/foutfeedback

## 2. Logica

### Scope

- Domeinverwerking en normalisatie
- Import parser en chunkverwerking
- Undo mechanisme voor delete
- Centrale servicefuncties zonder UI-afhankelijkheid

### Taken

- [x] Domein normalisatie centraliseren (lowercase, trim, IDN)
- [x] Servicefunctie voor add/update/delete maken
- [x] Import parser herbouwd met valid/invalid resultset
- [x] Delete met tijdelijke restore-key implementeren
- [x] Foutafhandeling uniform maken voor admin en AJAX

### Klaar wanneer

- [x] Logica is testbaar zonder directe UI-calls
- [x] Alle mutaties lopen via dezelfde service laag

## 3. Datamodel

### Scope

- Dedicated tabel voor geblokkeerde domeinen
- Unieke constraints en indexering
- Compatibele migratie op activatie

### Taken

- [x] Tabeldefinitie herzien en documenteren
- [x] Activatiepad via `dbDelta` herinrichten
- [x] Uninstallstrategie kiezen en eenduidig maken
- [x] Querymethodes met prepared statements nalopen
- [ ] Paginatie- en zoekqueries optimaliseren

### Klaar wanneer

- [ ] Schema lifecycle (activate/update/uninstall) is eenduidig
- [ ] Duplicaten zijn technisch onmogelijk door DB-constraint

## 4. Validaties

### Scope

- Domeinvalidatie
- Registratieblokkering en profielupdateblokkering
- Server-side validatie van alle input

### Taken

- [x] Domeinvalidatie in aparte helpermodule plaatsen
- [x] Regels voor labels/lengte/tekens volledig afdekken
- [x] Registratiehook en profielhook op top-level registreren
- [x] Eenduidige foutcodes en foutteksten gebruiken
- [ ] Edge cases toevoegen (IDN, lege input, malformed domeinen)

### Klaar wanneer

- [ ] Ongeldige domeinen worden consequent geweigerd
- [ ] Registratie met geblokkeerd domein wordt altijd geblokkeerd

## 5. UI Admin (instellen en aanpassen van geblokkeerde domeinen)

### Scope

- Duidelijke beheerinterface met snelle acties
- Import/export workflow
- Undo en bulk acties met bevestiging

### Taken

- [x] Pagina-opbouw herontwerpen met duidelijke secties
- [x] Formulier voor direct domein toevoegen
- [x] Inline bewerken vanuit de tabel
- [x] Bulk delete en delete-all-matching flow verbeteren
- [ ] Import preview met valid/invalid split tonen
- [x] Export knoppen toevoegen voor lijst en invalid regels
- [x] Undo notificatie met herstelactie tonen

### Klaar wanneer

- [x] Beheerder kan alle domeinacties uitvoeren zonder handmatig SQL werk
- [ ] Import van grotere lijsten werkt stabiel en zonder timeouts

## 6. UI Front End (en alles wat erbij hoort)

### Scope

- Duidelijke foutmelding voor eindgebruiker bij geblokkeerd domein
- Eventuele publieke informatiecomponent (optioneel shortcode)
- Integratie met registratie- en profielflows

### Taken

- [x] Registratiefoutmelding gebruiksvriendelijk formuleren
- [x] Profielupdatefoutmelding consistent maken
- [x] Teksten vertaalbaar maken via text-domain
- [ ] Front-end gedrag testen met standaard WordPress registratie
- [x] Optioneel: shortcode voor publieke uitlegpagina ontwerpen

### Klaar wanneer

- [x] Eindgebruiker begrijpt direct waarom registratie wordt geblokkeerd
- [x] Front-end meldingen zijn consistent en vertaald
